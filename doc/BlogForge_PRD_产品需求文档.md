# BlogForge 产品需求文档 (PRD)

**版本**: 1.0.0  
**日期**: 2026-05-08

---

## 1. 文档概述

### 1.1 文档目的

本文档旨在全面描述 BlogForge 桌面端博客管理软件的产品需求，为开发团队提供清晰的功能规格和技术约束，确保产品开发方向的准确性和一致性。

### 1.2 产品定位

BlogForge 是一款面向个人用户的跨平台桌面博客管理软件，提供从内容创作、本地预览到静态网站发布的完整解决方案。产品核心理念是"简单高效、SEO 友好、完全掌控"。

### 1.3 目标用户

- 个人博主、技术写作者
- 希望自主管理博客内容的创作者
- 对 SEO 有一定要求的网站运营者
- 偏好 Markdown 写作的开发者群体

---

## 2. 产品概览

### 2.1 核心价值主张

| 价值维度 | 具体描述 |
|---------|---------|
| 简单易用 | 直观的用户界面，零学习成本的 Markdown 编辑体验 |
| SEO 优化 | 自动生成 sitemap、RSS、robots.txt，支持自定义 Meta 信息 |
| 完全掌控 | 数据本地存储，支持多种发布平台，无厂商锁定 |
| 跨平台 | 支持 Windows、macOS、Linux 三大主流操作系统 |
| 高效发布 | 一键生成静态网站并部署到多种平台 |

### 2.2 技术架构概览

| 技术层面 | 技术选型 |
|---------|---------|
| 桌面框架 | Tauri 2.x + React/Vue |
| 本地数据库 | SQLite |
| 静态生成 | 自研轻量级静态网站生成器 |
| 编辑器 | Markdown 编辑器（支持代码高亮、数学公式、流程图） |
| 图床集成 | GitHub、阿里云 OSS、腾讯云 COS、Cloudflare R2、自定义 |

## 3. 功能需求

### 3.1 文章管理模块

#### 3.1.1 文章创建与编辑

- 支持创建新文章，自动生成 Front Matter（标题、日期、标签、分类等）
- Markdown 编辑器支持：
  - 代码块语法高亮（支持主流编程语言）
  - 数学公式（LaTeX/KaTeX 语法）
  - 流程图/图表（Mermaid 语法）
  - 图片上传与管理（拖拽、粘贴、选择上传）
  - 实时预览（分屏显示）
- 文章状态管理：草稿、已发布、回收站
- 自动保存与手动保存

#### 3.1.2 文章组织

- 标签系统：支持多标签、标签云展示
- 分类系统：支持层级分类
- 文章搜索：全文搜索、按标签/分类/日期筛选
- 文章排序：按创建时间、修改时间、标题排序

#### 3.1.3 版本管理

- 自动保存历史版本（可配置保存数量）
- 版本对比（Diff 视图）
- 一键回滚到指定版本

#### 3.1.4 导入导出

- 批量导入 Markdown 文件
- 导出文章为 Markdown/HTML/PDF
- 支持从其他博客平台导入（WordPress、Hexo 等）

### 3.2 图片管理模块

#### 3.2.1 图床配置

- 支持多种图床服务：
  - GitHub（免费，适合公开图片）
  - 阿里云 OSS / 腾讯云 COS（国内访问快）
  - Cloudflare R2（免费额度大）
  - 自定义图床（支持 API 接入）
- 图床配置管理：添加、编辑、删除、测试连接
- 默认图床设置

#### 3.2.2 图片上传

- 拖拽上传
- 粘贴上传（剪贴板图片）
- 选择文件上传
- 上传进度显示
- 自动插入 Markdown 图片语法

#### 3.2.3 图片库管理

- 已上传图片列表展示
- 图片搜索与筛选
- 图片删除（同步删除图床文件）
- 图片引用统计

### 3.3 预览模块

- 内置实时预览：编辑器内嵌预览窗口，实时渲染 Markdown
- 外部浏览器预览：启动本地服务器，在系统浏览器中预览完整站点
- 预览主题切换
- 响应式预览（桌面/平板/手机视图）

### 3.4 静态网站生成模块

#### 3.4.1 生成器核心功能

- 自研轻量级静态网站生成器
- 支持模板系统（预设主题）
- 自动生成：
  - 首页、文章列表页、文章详情页
  - 标签页、分类页、归档页
  - 关于页、友链页等自定义页面
- 增量构建：仅重新生成变更的文章

#### 3.4.2 SEO 优化

- 自定义 Meta 信息：每篇文章可设置 title、description、keywords
- Open Graph 标签：支持社交媒体分享优化
- 结构化数据：自动生成 JSON-LD
- sitemap.xml：自动生成站点地图
- RSS/Atom 订阅：自动生成 feed.xml
- robots.txt：可配置爬虫规则
- Canonical URL 设置
- 语义化 HTML 结构

### 3.5 发布部署模块

#### 3.5.1 本地导出

- 导出静态文件到本地目录
- 支持自定义导出路径

#### 3.5.2 Git 部署

- 配置 Git 仓库信息
- 自动 commit 和 push
- 支持 GitHub Pages、GitLab Pages 等

#### 3.5.3 FTP/SFTP 上传

- 配置服务器连接信息
- 支持 FTP 和 SFTP 协议
- 增量上传（仅上传变更文件）
- 上传进度显示

#### 3.5.4 一键发布

- 配置发布流程：生成 → 上传 → 完成
- 发布前检查（文章状态、链接有效性等）
- 发布日志记录

### 3.6 站点配置模块

- 站点基本信息：名称、描述、作者、URL
- 主题设置：预设主题选择与切换
- 导航菜单配置
- 社交链接设置
- 评论系统配置（可选集成）
- 统计代码配置（Google Analytics 等）

### 3.7 系统设置模块

- 界面语言：中文/英文切换
- 主题模式：亮色/暗色主题切换
- 编辑器设置：字体、字号、行高、自动保存间隔
- 版本历史设置：保存数量上限
- 数据备份与恢复

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|-----|------|
| 应用启动时间 | ≤ 3 秒 |
| 文章列表加载 | ≤ 1 秒（100 篇文章以内） |
| Markdown 渲染 | 实时预览延迟 ≤ 200ms |
| 静态网站生成 | ≤ 5 秒（100 篇文章） |
| 内存占用 | ≤ 200MB（空闲状态） |
| 安装包大小 | ≤ 50MB |

### 4.2 兼容性需求

- 操作系统：Windows 10+、macOS 10.15+、主流 Linux 发行版
- 屏幕分辨率：最低 1280×720，推荐 1920×1080
- 生成的静态网站兼容主流浏览器（Chrome、Firefox、Safari、Edge）

### 4.3 安全需求

- 敏感信息加密存储（图床密钥、FTP 密码等）
- 本地数据安全（SQLite 数据库加密可选）
- 网络传输使用 HTTPS/TLS

### 4.4 可用性需求

- 界面简洁直观，符合用户心智模型
- 提供完善的操作引导和帮助文档
- 错误提示清晰，提供解决方案建议
- 支持键盘快捷键

### 4.5 可维护性需求

- 模块化架构设计
- 完善的代码注释和文档
- 支持插件扩展机制（后期）

---

## 5. 用户界面需求

### 5.1 整体布局

- 三栏式布局：侧边栏（导航）、列表区、编辑/预览区
- 可调整各区域宽度
- 支持全屏编辑模式

### 5.2 主题风格

- 支持亮色/暗色主题切换
- 主题自动跟随系统设置（可选）
- 简约现代的设计风格

### 5.3 核心界面

- 仪表盘：站点概览、最近文章、快捷操作
- 文章管理：文章列表、筛选、搜索
- 文章编辑：Markdown 编辑器、实时预览、元数据编辑
- 图片管理：图片库、上传、配置
- 站点设置：基本信息、主题、导航、SEO
- 发布中心：生成预览、部署配置、发布日志

---

## 6. 数据需求

### 6.1 数据存储

- 本地 SQLite 数据库存储
- 文章源文件以 Markdown 格式存储
- 配置文件以 JSON/YAML 格式存储

### 6.2 数据备份

- 支持手动导出数据备份
- 支持从备份文件恢复
- 备份内容包括：数据库、文章源文件、配置文件、图片资源

---

## 7. 约束条件

### 7.1 技术约束

- 桌面框架：Tauri 2.x
- 前端框架：React 或 Vue 3
- 后端语言：Rust（Tauri 后端）
- 数据库：SQLite

### 7.2 业务约束

- 面向个人用户，无需多用户/权限管理
- 支持小型博客规模（100 篇文章以内为优化目标）

---

## 8. 交付物

1. 可运行的桌面应用程序（Windows、macOS、Linux 安装包）
2. 完整的源代码
3. 技术文档（架构设计、API 文档）
4. 用户手册
5. 预设博客主题（至少 3 套）
