# BlogForge UI/UX 设计规范文档

**版本**: 1.0.0  
**日期**: 2026-05-08

---

## 1. 设计原则

### 1.1 核心设计理念

- **简约高效**：减少视觉噪音，突出核心功能，让用户专注于内容创作
- **一致性**：统一的视觉语言和交互模式，降低学习成本
- **可预测性**：操作结果符合用户预期，提供即时反馈
- **可访问性**：支持不同能力的用户，符合 WCAG 2.1 AA 标准

### 1.2 设计目标

| 目标 | 具体指标 |
|-----|---------|
| 启动速度 | 应用启动时间 ≤ 3 秒 |
| 操作效率 | 常用功能最多 3 次点击可达 |
| 学习成本 | 新用户 5 分钟内完成第一篇文章发布 |
| 视觉舒适 | 支持亮色/暗色主题，减少视觉疲劳 |
| 响应速度 | 界面响应时间 ≤ 100ms |

## 7. 布局规范

### 7.1 整体布局

- 三栏式布局：侧边栏（导航）、列表区、编辑/预览区
- 侧边栏：宽度 240px，可折叠至 64px
- 列表区：宽度 300-400px，可调整
- 主内容区：自适应剩余宽度

### 7.2 响应式断点

| 断点名称 | 宽度范围 |
|---------|---------|
| Compact | < 768px |
| Medium | 768px - 1024px |
| Expanded | 1024px - 1440px |
| Large | > 1440px |

### 7.3 窗口最小尺寸

最小窗口尺寸：1280 × 720 px

## 8. 动效规范

### 8.1 动效原则

- **目的性**：动效应传达意义，引导用户注意力
- **自然**：模拟物理世界的运动规律
- **克制**：避免过度动效，影响性能和体验

### 8.2 动效时长

| 动效类型 | 时长 |
|---------|------|
| 微交互（按钮悬停） | 100-150ms |
| 状态切换（展开/折叠） | 200-300ms |
| 页面过渡 | 300-400ms |
| 复杂动画 | 400-600ms |

### 8.3 缓动函数

| 缓动类型 | 使用场景 |
|---------|---------|
| ease-out | 进入动画（展开、出现） |
| ease-in | 退出动画（折叠、消失） |
| ease-in-out | 状态切换动画 |
| linear | 持续动画（加载进度） |

## 9. 页面设计

### 9.1 仪表盘页面

- 顶部：欢迎语 + 快捷操作按钮
- 统计卡片：文章总数、已发布、草稿、访问量
- 最近文章列表：显示最近 5 篇文章
- 快捷入口：新建文章、预览站点、发布

### 9.2 文章列表页面

- 顶部工具栏：搜索框、筛选器、新建按钮
- 筛选标签：全部、已发布、草稿、回收站
- 文章列表：卡片式或列表式，显示标题、摘要、状态、日期
- 批量操作：选中后显示批量操作栏

### 9.3 文章编辑页面

- 顶部工具栏：保存、发布、预览、设置按钮
- 标题输入：大字号，突出显示
- 编辑区域：左侧 Markdown 编辑，右侧实时预览
- 侧边栏：文章设置（标签、分类、SEO、封面）
- 底部状态栏：字数统计、保存状态、版本历史

### 9.4 设置页面

- 左侧导航：设置分类（站点、主题、图床、部署、通用）
- 右侧内容：对应设置项表单
- 底部操作栏：保存、重置按钮

## 10. 交互规范

### 10.1 反馈机制

| 操作类型 | 反馈方式 | 示例 |
|---------|---------|------|
| 保存 | Toast 提示 + 状态栏更新 | 保存成功 |
| 删除 | 确认弹窗 + Toast 提示 | 确认删除？→ 已删除 |
| 上传 | 进度条 + 完成提示 | 上传中... → 上传成功 |
| 发布 | 进度弹窗 + 结果提示 | 发布中... → 发布成功 |

### 10.2 确认操作

以下操作需要二次确认：

- 删除文章（移入回收站）
- 永久删除
- 清空回收站
- 重置设置
- 删除图床配置

### 10.3 快捷键

| 快捷键 | 功能 | 适用场景 |
|-------|------|---------|
| Ctrl/Cmd + S | 保存 | 编辑器 |
| Ctrl/Cmd + P | 预览 | 编辑器 |
| Ctrl/Cmd + Shift + P | 发布 | 编辑器 |
| Ctrl/Cmd + N | 新建文章 | 全局 |
| Ctrl/Cmd + F | 搜索 | 列表页 |
| Ctrl/Cmd + , | 打开设置 | 全局 |
| Esc | 关闭弹窗/取消 | 全局 |

## 11. 无障碍设计

### 11.1 键盘导航

- 所有交互元素可通过键盘访问
- Tab 键顺序符合逻辑
- 焦点状态清晰可见

### 11.2 屏幕阅读器支持

- 所有图标添加 aria-label
- 图片添加 alt 文本
- 表单元素关联 label
- 动态内容使用 aria-live

### 11.3 颜色对比度

- 正文文字对比度 ≥ 4.5:1
- 大号文字对比度 ≥ 3:1
- 不仅依赖颜色传达信息
